# 开源部署检查清单

## 必须确保不提交

```gitignore
.env
state.json
*.log
memory_data/
profiles/my-profile.json
local_*.py
*_private.py
feishu_login_qr.png
.uploads/
```

## profile 模板化

- 公开仓库只放 `profiles/default.json` 和 `profiles/example-couple.json`
- 真实人设放在 `profiles/my-profile.json`（加入 `.gitignore`）

## 本地扩展

本地私有逻辑通过环境变量加载：

```env
LOCAL_DAILY_JOB_MODULE=local_daily_job
```

扩展文件 `local_daily_job.py` 已在 `.gitignore` 中。

## Dry Run

默认 `DRY_RUN=true`，部署后确认工作正常再改为 `false`。

## 上线前检查

```bash
make test
make build
go run ./cmd/doctor -online
git diff --check
```

macOS 上的 `make build` 会同时编译 Apple Vision OCR 助手。Linux 部署会跳过该步骤，并通过飞书 OCR 兜底。

## 数据目录

- `memory_data/<PROFILE_ID>/media/`：受管图片，内容哈希去重
- `memory_data/<PROFILE_ID>/audits/`：记忆清洗和检索评测报告
- `memory_data/<PROFILE_ID>/memories.json`：未配置数据库时的本地记忆

这些目录都不得提交到公开仓库。迁移机器时应使用加密备份，并单独迁移 `.env`。
