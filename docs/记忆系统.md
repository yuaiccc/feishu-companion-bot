# 记忆系统

记忆系统默认使用本地 JSON 文件，按 profile 隔离：

```text
memory_data/<PROFILE_ID>/memories.json
```

`memory_data/` 不应提交到 Git。

## 可见性

- `private`：绝不进入回复 prompt。
- `owner_only`：只在回复 owner 时可用。
- `public_to_target`：可以在回复目标用户时使用。

所有检索结果在进入 prompt 前必须做可见性过滤。

## 写入流程

普通群聊消息不能靠关键词直接写入长期记忆。正确流程：

1. DeepSeek 判断消息是否值得进入长期记忆候选。
2. 候选记忆通过私聊发给 owner。
3. owner 点“记住”才写入。
4. owner 点“不要记”则忽略。

## 检索流程

推荐流程是 agentic RAG：

1. embedding 召回候选。
2. 关键词分数兜底。
3. 模型根据当前问题和可见性重排。
4. `context_manager` 按预算拼入上下文。

默认 embedding 可以使用本地 hash。机器上有 Ollama 时，可以启用本地模型，例如：

```bash
ollama pull qwen3-embedding:0.6b
```

```env
MEMORY_EMBEDDING_PROVIDER=ollama
MEMORY_OLLAMA_EMBED_MODEL=qwen3-embedding:0.6b
```
