# 安全说明

## 隐私边界

- `.env`、运行日志、记忆库、聊天归档、媒体资产、私有 profile 和评测报告均为本地数据，不应提交到仓库。
- 受管图片存放在 `MEMORY_MEDIA_VAULT`，使用 SHA-256 去重；目录权限为 `0700`，文件权限为 `0600`。
- 记忆检索先按 `profile_id` 和可见性过滤，再拼接到模型上下文。`owner_only` 不会返回给目标用户或其他群成员。
- 发送给外部模型的消息经过敏感信息脱敏。本地 OCR、Ollama embedding 和本地视觉模型不会主动上传图片。
- DeerFlow、OpenClaw、文档评论、聊天归档和图片归档均需显式开启。

## 已知边界

- 数据库账号一旦拥有归档表权限，仍可在数据库层读取原始数据；部署时应使用最小权限账号。
- 大模型的判断不是访问控制。身份、profile 和可见性过滤必须由代码执行，不能只依赖提示词。
- 删除数据库记录不等同于擦除备份。需要同时制定媒体目录、数据库快照和日志的保留策略。
- `memory_data` 的本机文件权限不能替代磁盘加密；处理真实私人资料时建议开启 FileVault。

## 漏洞报告

请勿在公开 Issue 中粘贴令牌、聊天内容、OpenID、数据库连接串或私人图片。报告漏洞时只提供最小复现，并先撤销已经暴露的凭证。
